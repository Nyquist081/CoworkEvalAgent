🧮 TTTEC 六维评分模型 & 新增模块

基于补充契约的完整评分体系

📏 BaselineEvaluator — TTTEC 六维客观分
T1 · 任务完成度

Pandas 客观比对 50%
+ LLM 意图分 50%

T2 · 工具准确率

success / total × 100
无调用→默认100

T3 · 工具效率

100−5×超标次数
每次冗余扣5分

T4 · 思考效率

Token超标率×30
+ 轮次超标×5

E · E2E性能

超时比例映射扣分
max(0,100−超标%)

C · 成本效率

超支比例映射扣分
max(0,100−超支%)

🔬 src/evaluator/ResultComparator
  • Pandas 读取 output + reference 文件
  • 按 eval_config.ignore_rules drop 列
  • 行/列/数值 diff → T1 客观基础分
  • 禁止 LLM 直接读表格文件
📊 src/services/MetaAnalyzer
  • pass@k: k次中≥1次及格→pass
  • pass^k: k次全部及格→pass
  • 按 question_id 折叠计算
  • 输出 PP 差值 → 稳定性指标
⚖️ 最终融合公式
T1_final = Pandas客观分 × 50% + LLM意图分 × 50%

总分 = avg(T1, T2, T3, T4, E, C)

LLM 裁判只看 Trace 思考日志，不看表格数据
